# React Native Production Workflow

A comprehensive 15-phase workflow for building production-ready React Native applications from scratch.

## 📁 Structure

```
react-native-workflow/
├── phases/          # Step-by-step development phases
├── skills/          # Required developer skills
├── agents/          # AI agent configurations
├── hooks/           # Reusable custom hooks
└── rules/           # Coding standards & guidelines
```

## 🚀 Phases Overview

| Phase | Topic | Description |
|-------|-------|-------------|
| 01 | Project Planning | Requirements, architecture, tech stack |
| 02 | Environment Setup | Project initialization, tooling |
| 03 | Navigation | React Navigation setup, deep linking |
| 04 | State Management | Zustand, React Query |
| 05 | UI Components | NativeWind/Tamagui styling |
| 06 | API Integration | Axios, interceptors, types |
| 07 | Offline-First | AsyncStorage, sync, caching |
| 08 | Native Modules | Camera, location, notifications |
| 09 | Testing | Unit, integration, E2E tests |
| 10 | Performance | Optimization strategies |
| 11 | Security | Auth, encryption, best practices |
| 12 | Analytics | Crash reporting, tracking |
| 13 | App Store Prep | Icons, metadata, screenshots |
| 14 | CI/CD | Automated builds, deployments |
| 15 | Post-Launch | Monitoring, updates, feedback |

## 🎯 Getting Started

1. Start with **Phase 1: Project Planning**
2. Follow phases sequentially
3. Complete all deliverables in each phase
4. Review best practices before moving forward

## 🛠️ Tech Stack Recommendations

- **Framework**: Expo SDK 52+ or React Native CLI
- **Language**: TypeScript
- **Navigation**: React Navigation v6+
- **State**: Zustand + TanStack Query
- **Styling**: NativeWind (Tailwind) or Tamagui
- **Testing**: Jest + Detox
- **Backend**: REST/GraphQL with Axios

## 📝 Usage

Each phase document contains:
- Overview and objectives

Follow the phases in order for a structured development process.
